Enterprise AI architecture review

Critical review of Quatta AI architecture

This assessment reviews component placement, arrows, flow logic, missing controls, RAG design, connector design and agentic AI readiness for logistics, warehousing and retail customers.

Markdown review Stable architecture map

Truthful Verdict

The architecture is strong as a customer-facing blueprint. It is not yet a full production implementation spec without connector lifecycle, queueing, private connectivity, eval operations, data classification and DR details.

Robustness

It is not too robust conceptually. The needed simplification is clearer separation between request flow, data flow, control flow and outbound action flow.

Keep Unchanged

Keep read-only connectors, structured retrieval for operational truth, Context Builder, human approval gates and auditability.

01 Core platform

High-Level Quatta Architecture

Correct

  • Good platform, AI, customer and control separation.
  • Customer systems remain authoritative.
  • Context Builder belongs before agent reasoning.

Incorrect / Misleading

  • AI Gateway must wrap tool calls and response validation, not just pre-retrieval routing.
  • Security band must not read as passive monitoring.

Add / Improve

  • Flowchart drilldowns from high-level nodes.
  • Tool registry/MCP boundary.
  • Evaluation harness in implementation docs.
Updated relationship: User -> App -> Auth/RBAC -> AI Gateway policy -> Connector/Retrieval tools -> Context Builder -> Agent -> AI Gateway validation -> Grounded Response -> Audit.
02 Data access

Connector Layer Flow

Correct

  • Read-only source posture.
  • Authentication, normalization, freshness and audit are present.
  • Canonical output before AI use.

Incorrect / Misleading

  • Live query, sync, file drop and document ingestion patterns need separation.
  • Connector Modules must route to Data Access, not Core Platform.

Add / Improve

  • Connector registry, schema registry, retries, DLQ, idempotency and rate limits.
  • Credential rotation and source-owner escalation.
Updated relationship: Source -> Authenticate -> Connector Module -> Normalize -> Validate -> Cache or Normalized Data -> Context Builder -> Approved Agent Tool.
03 Retrieval

Retrieval + RAG Flow

Correct

  • Structured facts stay separate from document RAG.
  • Document AI, chunking, embeddings, pgvector, ACLs and citations are right.
  • RAG is not used as transaction truth.

Incorrect / Misleading

  • GCP/local storage staging was missing.
  • pgvector can appear to be the full retrieval layer.

Add / Improve

  • GCS staging, local/SFTP/API document entry paths.
  • Hybrid search, reranking, citation validation and retrieval evaluation.
  • No-answer behavior when evidence is insufficient.
Updated relationship: Question -> Retrieval Planner -> Structured Retrieval + Document Retrieval -> GCS Staging -> Document AI -> Chunking -> Embeddings -> pgvector -> Context Builder -> RAG Prompt -> Answer Validator.
04 Controls

Security, Governance And Audit

Correct

  • Identity, role mapping, tenant scope, permission decisions and human gates are present.
  • Policy version and audit event are correct controls.

Incorrect / Misleading

  • Filtered retrieval belongs inside Quatta, not in customer systems.
  • Revocation must cover cache, vectors, extracted text and answers.

Add / Improve

  • KMS, DLP/classification, immutable audit, break-glass admin and data residency.
  • Denied-path and cross-tenant tests in CI.
Updated relationship: Identity -> Token Validation -> Role Mapping -> Tenant Scope -> Permission Decision -> Retrieval Filter / AI Gateway Policy -> Human Gate -> Immutable Audit + Telemetry.
05 Run

Deployment And Operations Flow

Correct

  • Cloud Run, workers, Secret Manager, Cloud SQL, Cloud Storage, Vertex AI and Document AI are appropriate.
  • Dashboards, alerts, runbooks and rollback are necessary.

Incorrect / Misleading

  • The diagram is too optimistic if it stops at managed services.
  • CI/CD needs IaC, migrations, security scans and environment promotion.

Add / Improve

  • VPC/private connectivity, Cloud Tasks/Pub/Sub, HA/PITR, backup restore tests and SLOs.
  • Quota, cost and provider outage handling.
Updated relationship: GitHub -> CI Checks -> IaC Plan/Security Scan -> Build -> Deploy Approval -> Runtime Services -> Observability/SLO -> Rollback/DR.
06 Rollout

Customer Onboarding Lifecycle

Correct

  • Discovery, access design, source inventory, platform setup and evaluation are in the right order.
  • Agents come after connectors and retrieval.

Incorrect / Misleading

  • Enterprise legal, security and change-management gates need more weight.
  • Agent MVP should wait for golden-case acceptance.

Add / Improve

  • DPA/DPIA, data classification, private connectivity signoff, UAT and support handover.
  • SLA/SLO agreement before scale.
Updated relationship: Discovery -> Legal/Security/Data Classification -> Connectivity/RBAC -> Connector Build -> Retrieval + Eval Harness -> Agent MVP -> UAT -> Pilot -> Scale.
Example agent

Inventory Agent Flow

Correct

  • Deterministic reconciliation before model explanation.
  • No writeback by default.
  • Confidence and citations are included.

Incorrect / Misleading

  • Pattern Tool must be deterministic, not open-ended agent reasoning.
  • Financial impact must be role-gated.

Add / Improve

  • Snapshots, cycle counts, reservations, ASN/receipts, returns, lot/batch/serial, bins, ownership and ATP impact.
  • UOM conversion and transaction matching.
Updated relationship: Question -> Intent -> Permission -> Evidence Fetch -> Reconcile -> Deterministic Pattern + Financial Tools -> LLM Explanation -> Recommended Actions -> Manager Gate.
Example agent

Claims Agent Flow

Correct

  • Eligibility, liability, drafting, human review and approved send are separated.
  • Human approval is required before outbound action.

Incorrect / Misleading

  • Drafting must not imply sending.
  • Evidence pack and evidence summary are different components.

Add / Improve

  • Claim taxonomy, limitation periods, Incoterms/carrier SLA, duplicate check and deadline calculator.
  • Attachment security and outbound channel policy.
Updated relationship: Claim Intake -> Permission -> Ticket/Order/Contract/Evidence Retrieval -> Claim Case Context -> Eligibility Tool -> Liability Tool -> Evidence Summary -> Draft -> Human Review -> Approved Send/Export.
