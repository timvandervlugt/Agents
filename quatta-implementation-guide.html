Build guide for the Quatta AI layer

This page connects the architecture flowcharts to concrete setup work. It defines what to build, which services to use, when to use MCP, RAG, Vertex AI, Document AI and LangGraph, and what a production-ready customer pilot must prove.

Full overview doc Governance doc Roadmap doc
MCPUse for standardized agent tools with schemas, permission checks and audit logs. Keep connector logic reusable outside MCP.
RAGUse when answers depend on contracts, SOPs, claims evidence, PDFs, scans or policies. Structured ERP/WMS data should stay structured.
Vertex AIUse Gemini as the default enterprise model provider on GCP. Keep an OpenAI/Azure OpenAI provider adapter for customer preference.
Human gatesAgents recommend and draft. Humans approve claim sending, operational corrections and high-risk actions.

Services and when to use them

ServiceUse it forCritical setup detail
FastAPIQuatta API, AI Gateway, connector endpoints and agent endpoints.Use typed request/response models and structured error contracts.
Cloud RunStateless backend hosting and worker-style ingestion services.Separate runtime service accounts by API, ingestion and export workloads.
Microsoft Entra IDUser authentication and enterprise group/app-role mapping.Map Entra identity to Quatta tenant, role, site, customer and item permissions.
PostgreSQLAudit, config, normalized metadata, cache, human review state.Enforce tenant filters in every table and query path.
pgvectorVector index for SOPs, contracts, evidence and document chunks.Store tenant, ACL, document version and source timestamp with every chunk.
Vertex AI GeminiPrimary model inference for grounded answers and agent reasoning.Route all model calls through the AI Gateway, never directly from tools.
Document AIOCR for scans, PDFs, contracts, PODs and claims evidence.Preserve document ID, version, ACL and source timestamp through indexing.
LangGraphMulti-step workflows with branching, retries and human checkpoints.Persist state and audit every node execution.
MCPAgent-callable connector tools with discoverable schemas.Use read-only tools by default and enforce tool policy per tenant and role.
OpenTelemetryTrace requests, connectors, retrieval, model calls and human review.Track latency, errors, connector health, retrieval quality and model cost.

Pilot readiness

What must be true before a customer pilot

Access control

  • Entra sign-in works.
  • Tenant and role filters pass tests.
  • No cross-tenant vector retrieval.

Source confidence

  • At least one structured source connected.
  • At least one document source indexed.
  • Freshness warnings are visible.

Agent quality

  • Known cases pass evaluation.
  • Low confidence escalates.
  • Answers include citations.

Operations

  • Audit reconstructs every answer.
  • Dashboards and alerts exist.
  • Runbooks are ready.
